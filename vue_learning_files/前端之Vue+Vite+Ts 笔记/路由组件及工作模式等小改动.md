# 与一般组件区别：
一般组件一般要自己写标签，比如`<person></person>`
路由组件靠规则渲染，就是[[路由入门]]里面的path和component
### 一般组件习惯于放在component的下面
## 路由组件一般都存在pages或者views文件夹
### VUE3中，当切换组件后，会卸载原有，再用再挂载

# Vue3路由工作模式
## 记不记得之前在创建路由的router/index.ts里面有一个const Router?
```ts
const router = createRouter({
    history:createWebHistory(),
    routes:[
        {
            path:'',
            component:,
        },
    ]
})
```
### 其中这是history模式，
#### 优点是URL美观
#### 缺点是项目上线需要服务端处理路径，否则刷新404
## nginx里面解决，后端问题，解决很简单
hash模式就是URL网址带 /#/ 使得搜索等性能略差
```ts
history:createWebHashHistory(),
```
