# *路由传参比较*
- **`query` 优点**：
    - 灵活性高，可以传递多个参数。
    - 可以动态修改参数而不改变路径结构。
- **`query` 缺点**：
    - URL 可能变得冗长且不够语义化。
    - 参数容易被忽略或误改。
- **`params` 优点**：
    - URL 更加简洁、语义化。
    - 适合表示资源的唯一标识。
- **`params` 缺点**：
    - 不能动态修改（除非重新导航到新路径）。
    - 参数数量和类型有限制。（**不能传递对象和数组**）
