## 一个文件光是父目录就这么多，都是什么？
### **📁 文件夹**

1. **`.vscode`**  
    Visual Studio Code 编辑器的工作区配置，可能包含调试配置、插件推荐等。
2. **`node_modules`**  
    存放项目所有依赖的 npm 包（通过 `npm install` 或 `yarn install` 自动生成）。
3. **`public`**  
    静态资源目录，存放无需构建的文件（如图标、字体、`robots.txt`），直接复制到构建结果中。
4. **`src`**  
    核心代码目录，包含：
    - Vue 组件（`.vue` 文件）
    - 路由（`router`）
    - 状态管理（`store`，如 Pinia）
    - 入口文件（`main.ts`）

### **📄 文件**

1. **`.gitignore`**  
    定义 Git 版本控制需要忽略的文件/文件夹（如 `node_modules`、`.env` 等）。
2. **`env.d.ts`**  
    TypeScript 环境类型声明文件，用于声明全局类型或解决模块导入的类型问题（如 `.vue` 文件类型）。
3. **`index.html`**  
    应用入口 HTML 文件，Vue 根组件会挂载到此文件中的 `<div id="app"></div>`。
4. **`package.json`**  
    项目配置文件，记录依赖项、脚本命令（如 `dev`、`build`）、项目元数据等。
5. **`package-lock.json`**  
    锁定依赖版本号，确保不同环境下安装完全一致的依赖（由 npm 自动生成）。
6. **`tsconfig.json`**  
    TypeScript 的通用配置，定义编译选项、文件包含规则等。
7. **`tsconfig.app.json`** 和 **`tsconfig.node.json`**
    - `tsconfig.app.json`：针对前端代码的 TypeScript 配置（继承自 `tsconfig.json`）。
    - `tsconfig.node.json`：针对 Node 环境代码（如 Vite 配置文件）的 TypeScript 配置。
8. **`vite.config.ts`**  
    Vite 构建工具的配置文件，用于配置插件、服务器、构建选项等（如代理、别名 alias）。
9. **`README.md`**  
    项目说明文档，通常包含开发指南、部署步骤等。
