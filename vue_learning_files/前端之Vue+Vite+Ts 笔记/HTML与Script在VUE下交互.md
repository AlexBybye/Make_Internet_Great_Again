# 4.pinia的storetoRefs
### 管理全局状态
- **共享状态**：在大型应用中，多个组件往往需要访问和修改相同的状态。通过 Pinia 的 `store`，这些状态可以被集中管理，所有需要的组件都可以访问和修改它。
- **避免 prop drilling**：在组件层级较深的情况下，`store` 可以直接在任何组件中使用，无需通过父组件一层层传递。
    

### 状态的响应式

- **响应式数据**：Pinia 的 `store` 中的状态是响应式的。当状态发生变化时，所有使用该状态的组件都会自动更新。这保证了数据的一致性和视图的实时性。
    
- **高效更新**：Pinia 会跟踪状态的变化，并只更新受这些变化影响的组件，提高了应用的性能。
    

### 状态的组织和模块化

- **模块化管理**：可以将不同的状态和逻辑划分为多个 `store`，使代码更具模块化和可维护性。例如，可以将用户相关的状态放在一个 `store` 中，产品相关的状态放在另一个 `store` 中。
    
- **可重用性**：定义好的 `store` 可以在多个地方重用，提高代码的复用性。
    

### 状态的持久化和缓存

- **方便持久化**：可以轻松地将 `store` 中的状态持久化到本地存储（如 localStorage），实现页面刷新后状态的保持。
    
- **缓存管理**：可以实现对 API 请求结果的缓存，避免重复请求，提高应用的响应速度。
